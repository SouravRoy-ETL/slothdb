Auto-detection

SlothDB detects file format by extension. SELECT * FROM 'data.csv' just works.

Glob patterns

Query multiple files: read_csv('logs/2024-*.csv')

Virtual views on files

Create views on CSV/Parquet/Excel files. Always re-reads fresh data: CREATE VIEW sales AS SELECT * FROM read_csv('sales.csv')

slothdb 
-- Query a CSV file
SELECT department, COUNT(*), ROUND(AVG(salary))
FROM 'employees.csv'
GROUP BY department
ORDER BY AVG(salary) DESC;

-- Query Parquet with filter pushdown
SELECT * FROM read_parquet('events.parquet')
WHERE event_date > '2024-01-01';

-- Convert CSV to Parquet
COPY (SELECT * FROM 'big.csv')
TO 'big.parquet' WITH (FORMAT PARQUET);

-- Top earner per department (QUALIFY)
SELECT name, department, salary
FROM 'employees.parquet'
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY department
    ORDER BY salary DESC
) = 1;

Why SlothDB

Everything you need. Nothing you don't.

A production-grade analytical engine with zero external dependencies.

⚡

GPU Acceleration

CUDA (NVIDIA) and Metal (Apple Silicon). Aggregation, sorting, and filtering offloaded automatically when data exceeds 100K rows.

📋

130+ SQL Features

JOINs, window functions, CTEs (recursive), QUALIFY, MERGE, subqueries, set operations, 70+ built-in functions.

🚀

Vectorized Engine

Columnar execution processing 2,048 values per batch. Morsel-driven parallelism across all CPU cores.

💾

Single-File Persistence

Save your database to a .slothdb file. Atomic checkpoint with write-then-rename. Crash-safe.

📦

Compression + Zone Maps

RLE, dictionary, bitpacking codecs with automatic selection. Zone maps skip non-matching row groups.

🔌

Stable Extension ABI

C ABI guarantees backward compatibility. Extensions built for v1.0 work on v2.0 and beyond. Zero maintenance.

Use it everywhere

CLI, Python, C/C++

One engine, multiple interfaces. Pick the one that fits your workflow.

terminal 
# Start an in-memory session
$ slothdb

# Or use a persistent database
$ slothdb analytics.slothdb

# One-liner queries
$ slothdb -c "SELECT COUNT(*) FROM 'sales.csv'"

# Interactive session
slothdb> SELECT * FROM 'employees.csv' LIMIT 5;
slothdb> CREATE TABLE sales AS SELECT * FROM read_parquet('sales.parquet');
slothdb> SELECT region, SUM(revenue) FROM sales GROUP BY region;
slothdb> .quit
app.py 
import slothdb

# Connect
db = slothdb.connect()                     # in-memory
db = slothdb.connect("analytics.slothdb")  # persistent

# Query files directly
result = db.sql("SELECT * FROM 'sales.csv' WHERE region = 'US'")

# Get a pandas DataFrame
df = result.fetchdf()

# Window functions, CTEs, QUALIFY — full SQL
result = db.sql("""
    SELECT name, department, salary
    FROM 'employees.parquet'
    QUALIFY ROW_NUMBER() OVER (
        PARTITION BY department ORDER BY salary DESC
    ) = 1
""")

# Context manager
with slothdb.connect("data.slothdb") as db:
    db.execute("CREATE TABLE t AS SELECT * FROM 'data.csv'")
    print(db.sql("SELECT COUNT(*) FROM t"))
main.c 
#include "slothdb/api/slothdb.h"

int main() {
    slothdb_database *db;
    slothdb_connection *conn;
    slothdb_result *result;

    slothdb_open("analytics.slothdb", &db);
    slothdb_connect(db, &conn);

    // Query a CSV file directly
    slothdb_query(conn,
        "SELECT region, SUM(revenue) "
        "FROM read_csv('sales.csv') "
        "GROUP BY region", &result);

    for (uint64_t r = 0; r < slothdb_row_count(result); r++)
        printf("%s: %s\n",
            slothdb_value_varchar(result, r, 0),
            slothdb_value_varchar(result, r, 1));

    slothdb_free_result(result);
    slothdb_disconnect(conn);
    slothdb_close(db);
    return 0;
}

Installation

Get started in seconds

Choose your platform. Zero dependencies required.

Linux / macOS

$ curl -fsSL https://raw.githubusercontent.com/
  SouravRoy-ETL/slothdb/main/install.sh | bash

Windows

Download slothdb.exe from the latest release.

Python

$ pip install slothdb

Ubuntu / Debian

$ sudo dpkg -i slothdb_0.1.0_amd64.deb

Build from Source

$ git clone https://github.com/
  SouravRoy-ETL/slothdb.git
$ cd slothdb
$ cmake -B build -DSLOTHDB_BUILD_SHELL=ON
$ cmake --build build --config Release

Arch Linux

$ makepkg -si  # using PKGBUILD
