How we built SlothDB

Same embedded model as DuckDB and SQLite — link it in, point SQL at files on disk. The defaults are different.

Simple

Install with pip install slothdb and you're running. Point SQL at a file — no CREATE TABLE, no COPY FROM, no extension to load first.

Read more

Feature-rich

130+ SQL features — joins, CTEs, recursive queries, window functions, QUALIFY, MERGE, aggregates, set operations. Seven file formats built in, no extensions needed.

Read more

Fast

Vectorized columnar engine, morsel-driven parallelism, typed streaming scans that decode directly into DataChunks. 1.1×–8.6× faster than DuckDB on every query.

Read more

Free

Released under the MIT license. Use in commercial products, embed in desktop apps, redistribute freely. Source on GitHub, developed in the open.

Read more

Portable

A single ~8 MB binary. Runs on Windows, Linux, macOS, across x86_64 and arm64. Links into Python, C, C++. No runtime dependencies, no JVM, no Python required for the CLI.

Read more

Extensible

Stable C ABI for third-party extensions. Numeric error codes that never change. Extensions built for v0.1 keep working on v1.0, v2.0, and beyond — unlike DuckDB's internal C++ API.

Read more

Install SlothDB

Single binary. No runtime dependencies. Pick a client.

Python

Full SlothDB engine as a pip-installable package. Connection API, pandas integration, context manager support.

pip install slothdb
python -c "import slothdb; slothdb.demo()"

Linux & macOS

Downloads the prebuilt CLI binary into ~/.local/bin. Works on x86_64 and arm64.

curl -fsSL \
  https://raw.githubusercontent.com/SouravRoy-ETL/\
  slothdb/main/install.sh | bash

Windows

Download slothdb.exe from the latest release. No installer — just unzip and run.

Build from source

Clone, CMake, and build. Produces a shared library, static library, and CLI binary.

git clone https://github.com/SouravRoy-ETL/slothdb
cd slothdb
cmake -B build -DSLOTHDB_BUILD_SHELL=ON
cmake --build build --config Release
