A fast in-process SQL database
for your local data files.

SlothDB is a drop-in DuckDB alternative. Point SQL at Parquet, CSV, JSON, Avro, Arrow or SQLite — no server, no import step, no extensions. 1.1×–8.6× faster than DuckDB on every query in the benchmark suite.

$ pip install slothdb
Get started View on GitHub Read the docs

Try it in 60 seconds

No files to find. The demo generates a 100 000-row CSV, runs three queries through SlothDB, and prints a side-by-side timing against DuckDB on your own machine.

Use it from any language

SlothDB links into your process. No server to start, no network. The same binary powers the Python package, the CLI, and C/C++ embedding.

import slothdb

db = slothdb.connect()                                  # in-memory
db = slothdb.connect("analytics.slothdb")              # persistent

# Query files directly — no CREATE TABLE, no COPY FROM
df = db.sql("""
    SELECT region, SUM(revenue)
    FROM 'sales.parquet'
    WHERE year >= 2023
    GROUP BY region
""").fetchdf()    # → pandas DataFrame
# Interactive shell
$ slothdb
slothdb> SELECT * FROM 'data.parquet' LIMIT 5;

# Or one-shot from the command line
$ slothdb -c "SELECT region, SUM(revenue) FROM 'sales.csv' GROUP BY region;"

# Persistent database
$ slothdb analytics.slothdb
#include <slothdb/api/slothdb.h>

slothdb_database *db;
slothdb_connection *conn;
slothdb_result *result;

slothdb_open("analytics.slothdb", &db);
slothdb_connect(db, &conn);
slothdb_query(conn,
  "SELECT region, SUM(revenue) FROM 'sales.csv' GROUP BY region",
  &result);

for (uint64_t r = 0; r < slothdb_row_count(result); r++)
  printf("%s: %s\n",
    slothdb_value_varchar(result, r, 0),
    slothdb_value_varchar(result, r, 1));

slothdb_free_result(result);
-- Parquet / CSV / JSON / Arrow — same syntax
SELECT region, SUM(revenue)
FROM 'sales.parquet'
WHERE year >= 2023
GROUP BY region;

-- Avro, Arrow, SQLite — no extension load needed
SELECT * FROM read_avro('events.avro');
SELECT * FROM sqlite_scan('app.db', 'users');

-- Export to any format
COPY (SELECT * FROM 'big.csv' WHERE year >= 2024)
TO 'filtered.parquet' WITH (FORMAT PARQUET);

-- QUALIFY for top-N-per-group without subqueries
SELECT name, department, salary
FROM 'employees.parquet'
QUALIFY ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) = 1;

Why SlothDB?

Same embedded model as DuckDB and SQLite. The defaults are different.

Simple

Point SQL at a file. No CREATE TABLE, no import, no extension to load. pip install slothdb and you're running.

Seven formats built in

Parquet, CSV, JSON, Avro, Arrow, SQLite and Excel — all in core. DuckDB needs extensions for Avro, SQLite and Excel.

Fast

1.1×–8.6× faster than DuckDB across 15 benchmark queries. Vectorized columnar execution, morsel-driven parallelism, typed streaming scans.

Portable

Single ~8 MB binary. Runs on Windows, Linux, macOS. Links into Python, C, C++. No runtime dependencies.

Stable C ABI

Numeric error codes (ErrorCode::TABLE_NOT_FOUND = 2000) that never change. Extensions built for v0.1 keep working on v1.0.

Free and open

MIT licensed. Source on GitHub. 326 tests, 131 000+ assertions. Use it anywhere, including in commercial products.

Ecosystem

One engine, many on-ramps.

File formats

Parquet (zstd/snappy), CSV (RFC 4180), JSON/NDJSON, Avro, Arrow IPC, SQLite, Excel (xlsx). All native readers — no plugins, no conversion step.

Python

Installable via pip. Connection API, fetchdf() returns a pandas DataFrame. Works in Jupyter, scripts, FastAPI, Dagster.

pypi.org/project/slothdb →

C / C++

Stable C ABI headers, CMake integration, RAII-friendly pointer lifecycle. Embed in desktop apps or server-side tools.

C/C++ API reference →

CLI shell

Interactive shell with line editing, .tables / .schema / .help meta-commands, persistent history, UTF-8 support.

Extensions

Build custom functions and table scans against a stable C header. Load at runtime via LOAD 'my_ext'. Won't break on version bumps.

Extension guide →

Dataframe interchange

Convert query results directly to pandas DataFrames. Arrow IPC read/write for zero-copy handoff to other tools.

Install

Single binary. No runtime dependencies.

Python

pip install slothdb
python -c "import slothdb; slothdb.demo()"

Linux / macOS

curl -fsSL \
  https://raw.githubusercontent.com/SouravRoy-ETL/slothdb/main/install.sh \
  | bash

Windows

Download slothdb.exe from the latest release.

Build from source

git clone https://github.com/SouravRoy-ETL/slothdb
cd slothdb
cmake -B build -DSLOTHDB_BUILD_SHELL=ON
cmake --build build --config Release
